# Архитектура ChatBot Backend - Task Chain Generator

## Обзор проекта
Данный бэкенд предназначен для обработки сложных запросов от пользователя, которые представлены в виде последовательности шагов (цепочки сообщений). Система должна уметь управлять взаимодействием с различными LLM API (OpenRouter, LM Studio) и применять паттерн "Стратегия" для переключения между ними.

## 📚 Текущая реализация (Current State)
Проект уже содержит базовые модули для:
1. **Управления ключами (`keys_manager`):** Реализована система получения, сохранения и управления API ключами через `KeyManager`, `EnvManager` и `KeyDistributor`. Поддерживается механизм отметки ключей как "занятых" (busy) и возможность их удаления.
2. **Логирования (`logger`):** Настроен асинхронный логгер для записи информации об ошибках и процессе работы в консоль и файл.
3. **Основной модуль:** `main.py` является точкой входа, но пока минимален.

## 🚀 Требуемая функциональность (Target Architecture)

### 1. Ядро системы: Task Chain Processor
Центральный компонент, который принимает JSON-объект с цепочкой задач (`chain`) и управляет последовательным выполнением шагов.

**Входной формат:**
```json
{
    "chain": [
        {"prompt": "...", "action": "generate/check/correct/consult", "model": "auto"},
        ... 
    ],
    "user-id": "...",
    "mode": "fast/deep/research"
}
```

**Логика обработки:**
*   Последовательное выполнение шагов.
*   Обработка различных `action` (generate, check, correct, consult).
*   Применение глобального режима (`fast`, `deep`, `research`) для модификации промптов на каждом шаге.

### 2. Модуль API-интеграции (LLM Service)
Отвечает за фактический вызов LLM. Должен быть реализован паттерн "Стратегия" для поддержки:
*   **OpenRouter:** Основной провайдер ключей и запросов.
*   **LM Studio:** Альтернативный или дополнительный бэкенд.

### 3. Модуль Обработки Процесса (Action Handlers)
Отдельные функции/классы для реализации логики каждого действия:
*   `generate`: Базовая генерация текста.
*   `consult`: Добавление инструкций-советов в промпт (Think Mode).
*   `check`: Запрос на проверку правильности и получение списка исправлений.
*   `correct`: Прием указаний и возврат полного, исправленного сообщения.

### 4. Модуль Управления Контекстом и Состоянием
Отвечает за передачу истории сообщений между шагами цепочки для поддержания контекста.

## 🛠️ План действий (Action Plan)

2.  **Создание `LLMService`:** Реализовать паттерн Стратегия для взаимодействия с OpenRouter и LM Studio.
3.  **Создание `TaskChainProcessor`:** Основной класс, который принимает цепочку и управляет выполнением шагов.
4.  **Реализация Action Handlers:** Написание логики для `generate`, `check`, `correct`, `consult`.
5.  **Интеграция:** Обновление `main.py` для приема входных данных и запуска процесса.
