# Архитектура ChatBot Backend - Task Chain Generator

## 📚 Обзор проекта
Данный бэкенд предназначен для обработки сложных запросов от пользователя, которые представлены в виде последовательности шагов (цепочки сообщений). Система должна уметь управлять взаимодействием с различными LLM API (OpenRouter, LM Studio) и применять паттерн "Стратегия" для переключения между ними.

## 🚀 Текущая реализация (Current State)
Проект включает следующие ключевые модули:

### 1. Управление Ключами (`keys_manager`)
*   **`KeyManager`**: Центральный класс, взаимодействующий с OpenRouter API для генерации, удаления и получения информации о ключах. Использует `EnvManager` для безопасного хранения мастер-ключа.
*   **Функционал:** Поддержка механизма отметки ключей как "занятых" (`busy`) и получение списка доступных ключей.

### 2. Ядро Обработки Цепочки (`core/task_chain_processor.py`)
*   **`TaskChainProcessor`**: Основной класс, который принимает JSON-объект с цепочкой задач (`chain`). Он управляет последовательным выполнением шагов и отвечает за:
    *   Применение глобального режима (`fast`, `deep`, `research`) к промптам.
    *   Вызов специализированных обработчиков действий (`generate`, `check`, `correct`, `consult`).
    *   Поддержание контекста истории сообщений между шагами.

### 3. Сервисы LLM (Паттерн Стратегия)
*   **`base_llm_service.py`**: Абстрактный базовый класс, определяющий контракт для всех LLM-провайдеров (`generate_content`, `check_content`, `correct_content`, `consult_context`).
*   **`openrouter_llm_service.py`**: Конкретная реализация, использующая OpenRouter API через метод `requests.post`. Обрабатывает контекст и режимы в соответствии с требованиями.
*   **`lmstudio_llm_service.py`**: Альтернативная реализация для локального запуска LLM (LM Studio), используя другой эндпоинт (`http://localhost:1234/v1/chat/completions`).

## 🎯 Требуемая функциональность и План действий (Target Architecture & Action Plan)
Цель — создать отказоустойчивую, масштабируемую систему обработки сложных задач.

### 1. API Documentation (Документация по API)
*   Создать отдельный файл документации для каждого сервиса LLM, описывающий входные и выходные данные методов (`generate_content`, `check_content` и т.д.).

### 2. Улучшение TaskChainProcessor
*   **Обработка ошибок:** Добавить более надежную обработку исключений на уровне процессора при сбое любого шага в цепочке.
*   **Выбор сервиса:** Внедрить механизм выбора LLM-сервиса (OpenRouter/LM Studio) на основе конфигурации или явного указания пользователя, используя паттерн Стратегия.

### 3. Модули и Интеграция
*   **`main.py`**: Обновить точку входа для инициализации `TaskChainProcessor`, выбора нужного LLM-сервиса и запуска процесса с входным JSON.
*   **Дополнительные модули:** Рассмотреть создание модуля для валидации входящего JSON-запроса (схема).
